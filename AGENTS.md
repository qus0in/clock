# 코드 및 깃 컨벤션 (Project Conventions)

## 0. 핵심 원칙 (Core Principles)
- **명시적 요청 필수**: 배포(Deploy) 및 원격 저장소 동기화(Sync/Push)는 사용자의 **명시적인 요청(슬래시 명령어 포함)**이 있을 때만 수행한다. 어떠한 경우에도 자동 푸시/배포를 수행하지 않는다.

## 1. Git Convention
- **커밋 메시지 형식**: `카테고리: 짧은 한글 설명`
- **카테고리 (영어)**: 전형적인 깃 컨벤션 준수 (예: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`)
- **작업 원칙**: 한 가지 작업을 마친 후 바로 커밋을 진행한다. (원격 푸시는 금지)

## 2. Code Convention
- **파일 길이**: 각 파일은 최대 80줄 이내로 작성한다.
- **모듈화**: 코드는 최대한 모듈화하여 관리한다. 기능 추가 등으로 인해 80줄을 초과할 것 같으면 공통 부분을 추출하여 외부 모듈(별도 파일)로 분리한다.
- **주석 규칙**: 
  - 주요 선언부와 흐름 제어 부분에 간략한 한글 주석을 작성한다.
  - JSDoc 스타일 (`/** ... */`) 은 사용하지 않는다. (일반 `//` 주석 사용 권장)

## 3. UI 및 성능(Architecture) 원칙
- **글래스모피즘(Glassmorphism)**: 새로운 UI 컴포넌트 추가 시, 기본적으로 그림자(`shadow`) 대신 얕은(반투명) 배경색, `backdrop-blur`, 얇고 투명한 `border`를 사용하여 글래스모피즘 질감을 유지한다.
- **FOUT/CLS (레이아웃 시프트) 억제**: 숫자가 지속적으로 변하는 시계와 같은 UI에서는 `font-mono` 또는 `tabular-nums` 속성을 적용해 변경 시 너비 흔들림(CLS)을 최소화한다. 또한 커스텀 외부 폰트는 `document.fonts.ready` Promise를 사용해 클라이언트 마운트 시점에서 모두 폰트 파일이 로딩된 뒤 그리도록 통제한다.
- **OpenNext 배포 캐싱 고려**: Cloudflare 배포 시, 시계와 같은 실시간성 페이지 단위는 컴포넌트 내부에서 `force-dynamic` 처리를 유지하고 브라우저 단위 캐싱(`_headers`) 정책을 깐깐하게 분리해 작성한다.
