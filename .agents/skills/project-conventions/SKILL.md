# Project Conventions

이 스킬은 프로젝트의 코드 및 깃 컨벤션을 강제하기 위해 사용됩니다. 코드를 작성하거나 수정할 때 항상 다음 규칙을 준수해야 합니다.

## Git Convention
1. 커밋 메시지는 `type: 한글 설명` 형식으로 작성합니다. (예: `feat: 로그인 컴포넌트 추가`, `fix: 스타일 오류 수정`)
2. 하나의 논리적인 작업이 끝나면 반드시 커밋을 진행합니다.
3. **절대** 사용자의 명시적인 요청 없이 원격 저장소(remote)에 코드를 푸시(push)하거나 배포(deploy)하지 마십시오. 배포와 싱크는 전용 워크플로우를 통해 사용자가 지시할 때만 수행합니다.

## Code Convention
1. **80줄 제한**: 모든 파일은 최대 80줄을 넘지 않도록 작성합니다.
2. **모듈화**: 기능 추가로 코드가 80줄을 초과할 경우 공통 부분을 외부 모듈로 즉시 분리하여 컴포넌트나 함수 단위로 분할합니다.
3. **주석 원칙**: 주요 선언부와 로직 제어 부분에 간결한 한글 주석을 `//` 형식으로 추가합니다. JSDoc 식의 `/** ... */` 구문은 사용하지 마십시오.
